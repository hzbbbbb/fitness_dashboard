# AI 接任务后的固定输出规范

## 目的

本文件用于约束 AI 在开始工作前、工作中、工作后的输出结构，降低“没读规则就动手”和“改完却没说清楚”的问题。

## 开始工作前必须输出

AI 在开始分析或修改前，必须按以下顺序输出：

1. 已阅读的 `doc` 文件
2. 对任务目标的理解
3. 当前判断涉及的规则文件
4. 计划修改的文件
5. 为什么要这样修改
6. 风险点和边界判断

## 分析阶段输出模板

推荐格式：

```text
已阅读文档：
- ...

任务理解：
- ...

适用规则：
- ...

计划修改：
- ...

风险点：
- ...
```

## 修改进行中输出要求

- 必须说明当前在看什么或改什么
- 必须说明这样做和任务目标的关系
- 不要只说“我来处理一下”
- 如果发现任务与规则冲突，必须马上指出

## 修改完成后输出模板

推荐格式：

```text
修改文件：
- ...

变更内容：
- ...

实现结果：
- ...

未完成项：
- ...

后续建议：
- ...
```

## 禁止输出

- 不要跳过“已阅读文档”直接开改
- 不要只给结果不给原因
- 不要把猜测写成事实
- 不要用空泛表述代替具体修改说明
