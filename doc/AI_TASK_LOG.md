# AI TASK LOG

## 说明

本文件用于记录“提示词驱动的开发过程”，不是 Git 提交记录，也不是自动生成的变更清单。

记录目标：

- 保留每次 AI 任务的背景和目标
- 追踪实际完成结果
- 明确尚未完成的后续事项
- 方便后续继续追加，而不是依赖上下文记忆

适用模块：

- `desktop`
- `mobile`
- `doc`

## 使用规则

- 每完成一轮明确任务，就新增一条记录
- 记录内容应以“任务”为单位，而不是以 commit 为单位
- 标题应概括这次任务目标
- “我给 AI 的目标”尽量保留原始意图，不要过度压缩
- “实际完成内容”只写已经落地的结果
- “后续待办”只写还没完成、但下一步可能继续推进的事项
- 日期统一使用 `YYYY-MM-DD`

## 记录模板

复制以下模板，直接追加到文件末尾：

```md
## YYYY-MM-DD - 任务标题

- 任务标题：
- 涉及模块：desktop / mobile / doc
- 我给 AI 的目标：
- 实际完成内容：
- 后续待办：
```
